# Учёт заказов в интернет-магазине услуг

Простое приложение на PyQt6 по Варианту 2 контрольной точки 4 («Учёт услуг»).

## Запуск

```bash
pip install -r requirements.txt
python main.py
```

Данные хранятся в `orders.csv` рядом с программой и перезагружаются
из файла после любого добавления, редактирования или удаления заказа.

## Структура проекта

- `main.py` — точка входа, главное окно со стеком из двух страниц.
- `data.py` — модель заказа, чтение/запись CSV, справочники для выпадающих списков.
- `list_page.py` — страница «Список заказов»: таблица с подсветкой строк, кнопка удаления, переход в редактирование по клику на строку.
- `form_page.py` — страница «Добавление / редактирование заказа» (одна форма на оба режима).
- `orders.csv` — файл с данными.

## Реализованные правила подсветки

- Статус «В работе» — светло-жёлтый фон (#FFFFE0).
- Статус «Выполнен» — светло-зелёный фон (#E0F7FA).
- Статус «Отменён» — светло-красный фон (#FFCDD2).
- Скидка > 20% — насыщенный зелёный фон (#2E8B57), приоритетнее цвета статуса.
- При наличии скидки базовая стоимость показывается перечёркнутой красным, итоговая — чёрным рядом.
- Способ оплаты «При получении» — маркер 💳→💵 рядом с названием.

## Проверки в форме

- Наименование услуги обязательно для заполнения.
- Дата выполнения не может быть раньше даты оформления.
- Базовая стоимость и количество — неотрицательные (ограничено диапазоном полей).
- Скидка — от 0 до 100%.
- Итоговая стоимость пересчитывается автоматически при изменении базовой стоимости/скидки и не может превышать базовую (поле программно ограничено по максимуму).
- № заказа недоступен для редактирования и не отображается при добавлении нового заказа.
